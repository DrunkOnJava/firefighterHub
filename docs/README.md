## Contributing to Documentation

When adding new documentation:

1. Use clear, descriptive filenames
2. Follow the existing naming conventions (UPPERCASE for major docs)
3. Update this README with links to new important documents
4. Move outdated/completed documentation to appropriate archive folders
5. Use markdown format with proper headings and structure

## Documentation Best Practices

- Keep documentation up-to-date with code changes
- Archive historical documents rather than deleting them
- Use relative links when referencing other docs
- Include code examples and screenshots where helpful
- Add a table of contents for longer documents
